Overview

The purpose of the integrated project is for our team to create a web application that will demonstrate how to access data through the use of API’s and then visualise the data. For those who are unaware on what an API is, it is essentially the user interface for machines. The easiest way to describe it would be comparing an API to a phone and the machine to a person where the machine will use an API to gain information just like how a person can use their phone to gain information.

In this web application the team was made aware that altogether there would be six webpages that would use API’s to gain data then visualise said data. The project spec has made it clear for the team what two of the webpages would entail however the other four web pages were up to the team's own imagination. The team was also made aware that each webpage would have a tutorial webpage linked to it and the tutorial webpage will briefly explain the visualisation techniques being used, aspects of the data contents, where to source the data and the API access.


The two webpages that the project spec specifies to create are the following:

Earthquake

When a user enters the earthquake page, they will see a map on the webpage that will remind them of google maps and this is due to It being the google maps map. On screen it will also show several buttons that will populate the map with different data markers to do with earthquakes. The markers that appear on the map represent an earthquake that has occurred.

For all the buttons that will be mentioned each one will have the functionality of showing clustered markers. This means that any marker that will overlap another or in close proximity to another will join together and the new marker will have a number on it that relates to the number of markers that are clustered. Users will be able to click on the clustered marker to zoom in on the map to un-cluster the marker.

Another feature that all buttons will have will be that when a single marker is shown the marker will have a number on it representing the magnitude of the earthquake.

The final feature that all buttons will have will be that when a single marker is clicked on an info window will appear. On this info window there will be a link that will open up a new page that when opened will populate the page with relevant info about the area of where the earthquake happened.

On the screen there are 5 buttons and 4 radio buttons. The 4 radio buttons are titled past hour, past day, past week and past 30 days. The 5 buttons are titled All earthquakes, 1.0+ Earthquakes, 2.5+ Earthquakes, 4.5+ Earthquakes and Significant. For the user to populate the map with markers they will first have to choose one of the radio buttons and then click on one of the five buttons. For example, a user choses Past Day and then clicks on All Earthquakes the map will then be populated with markers of All Earthquakes that have happened in the past day.

Weather

On this page the user will be able to view relevant weather information as well as a relevant whether image by either entering in the latitude and longitude of the location they wish to view, or the user will enter in the location name.

To do this the user will click on either the latitude and longitude button to unlock the latitude and longitude input fields or the user will click on the location button to unlock the location input field. When the user clicks on one of the buttons the opposite input fields will be locked so the user will be only able to enter latitude and longitude or the location and never be able to do both.

Once the user has entered their chosen input field, they will then hit the button submit to view a weather image relevant to the weather of the location they have entered as well as showing useful information relevant to the weather feed.

Java Script techniques

When coding this project, we will be using some JavaScript to create the web pages, within JavaScript there are things called JavaScript techniques. A JavaScript technique is a way of improving the readability of the code and the general performance and resource usage of the code.

One JavaScript technique that we used was the use of proper variable names, this is an important technique to make use of as it improves the readability of the code as using proper variable names makes it easier to follow code, debug the code and it makes it easier to understand what each variable stores as it will have a name appropriate to its use.

Another JavaScript technique that we have made use of for the code is, we used text indentation properly. This technique is important to the code readability as it helps to show where loops start and being using indentation as the start of a loop and the end will be less indented than the inside of the loop which means it is easier to follow and understand loops.

One other way of using JavaScript techniques to improve the readability of the code is to use comments throughout the development process. Through the use of comments we can see what each function does, what each variable is used for and what every class is for, this helps improve the readability of the code as using comments means that is easier to quickly find and maintain any problem areas of code as it is already described for us.

Browser developer tool is a technique that the team frequently used as this tool is most commonly used when a developer is debugging their code. With the help of this debugger tool the developers will be able to find where the main errors are with ease. This tool will be used mostly during the development phase as the developers will use this during their unit testing.

GeoJson

Within this website, we are required to make use of geographical data in order for some of the webpages to function. The two main pages that require geographical data for the API’S to run are the Earthquakes page and the Weather page. For these API’S to run and function correctly we will be using geographical data in the GEOJSON format.

GEOJSON is the standard format for displaying geographical types and data, the types that are supported by this format are, Points, LineString, Polygon, MultiPoint, MultiLineString and MultiPolygon.

Points are used to display a singular geometrical point, this is also the simplest form of geometrical data that GEOJSON will display as the data is only displayed in one point, and this could be used to identify 1 singular area of geographical interest.
LineStrings are used to display two points of geographical data that are connected through a line, this could be used to calculate the distance between 2 geographical areas of interest.
Polygons are the more complex GEOJSON data types, this is where we have 4 points that are mapped out to display a polygon shape, and this could be used to represent a large area of geographical interest such as the area with the most earthquakes in a month.
The data types that are multi are simply used to support more than one data entry.

For the Earthquakes page the API will read in the data and then display all the points on a map that the earthquakes have happened over a period of time, for this to function properly we will use GEOJSON multi points as we will display multiple points of geographical interest.

For the weather page we will also be using multipoint as the data type due to the pages API displaying the weather as a single point for that day.
