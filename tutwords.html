Synonym FinderTutorial

This following page will take you through step by step on how to use the API that we used effectively and how to use the visualisation for this page as well.

For this page, there is only one Ajax request however this Ajax request is inside a function it is done this way as it is most likely will be called multiple times.

An Ajax request main purpose is to update a webpage asynchronously, meaning a user will not have to refresh a webpage for a tiny change to take effect. There are four main features to an ajax request the first would be to update a webpage without reloading it, the second would be to request data from a server after the page has loaded up, the third would be to receive data after a page has loaded up and the forth would be to send data to a server and this would happen in the background.

To use an Ajax request we first need to implement jQuery. jQuery is a lightweight JavaScript library, this means that writing in JavaScript will become a lot easier as with this library implemented tasks that use to require a lot of lines of code to function now require less as the jQuery library will most likely have useful functions that the developer could use. The jQuery library will also help simplify the Ajax requests that will be seen throughout this website.

The API that this webpage uses is the following:
//api.datamuse.com/words?rel_syn+ sourceWord + &max= + max
For this API the rel_syn part of the call is what ensures that the data we receive back are synonyms. This part of the call can be changed so that the received data back could be anything relating to words, for example, the user could receive back popular adjectives by entering in rel_jjb instead. The sourceWord variable in the call is the word that a user will choose to find out what synonyms it has. The final variable needed to make this API call work is the max variable and this variable will determine how many synonym words we will receive back.

For this page specifically, the Ajax request is within a function that can only be called 3 different ways. The first time it will be called is when the webpage initially loads up and this will use the function to load our initial spider diagram with the starting word as start. The other two ways this function can be called upon is when the user clicks on the centre word or when a user clicks on a branched word.



The visualisation that is used for this page is a spider diagram. This visualisation is hard-coded into our webpage and we found the code through the example visualisation on the API web page. We took this visualisation and made it our own by changing the colour scheme and the shape and size of the boxes used.

When a new spider diagram is generated we use the function called response to first determine if the word has any synonyms if it does not then an error message will appear for the user. If there are synonyms for the word, then the spider diagram that was there, to begin with, will be erased. There will also be a statement in this function to fill the array called link with the synonym data. Later in the function, we use the for each method to the array to generate the new spider diagram.

The function response also determines the layout of the spider diagram. This would be the spacing in between each box and how far away it is from the user inputted word.
